这是博客的第一篇测试文章，用来验证 GitHub Pages 的发布流程、Jekyll 页面结构，以及文章页的基础排版效果。

当前站点采用 `Jekyll + GitHub Pages` 的最小方案，优先完成：

- 基础页面结构
- 文章发布流程
- 中文内容展示

## 为什么先用 Jekyll

当前目标是先把博客稳定上线，因此优先选择 GitHub Pages 原生支持较好的方案。这样可以先把精力放在写作和结构迭代上，而不是环境配置。

> 先让内容持续发布，再逐步增加功能，通常比一开始追求“完整功能栈”更稳妥。

## 当前站点结构（v1）

目前保留的核心页面有：

- 首页（展示博客定位与最近文章）
- 博客归档
- 分类索引
- 标签索引
- 关于页面
